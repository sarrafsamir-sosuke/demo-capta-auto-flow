# Client Brand Interface Designer

## Objetivo

Criar e refinar interfaces com base na identidade visual do cliente, evitando designs genericos, repetidos ou com cara de IA/template.

## Regra principal

Antes de definir o design, analisar as referencias visuais do cliente.

O design deve parecer uma extensao da marca do cliente, nao da Capta.

## Use quando

- O usuario pedir para criar ou melhorar um CRM.
- O usuario pedir landing page.
- O usuario pedir dashboard.
- O usuario pedir site.
- O usuario pedir sistema interno.
- O usuario pedir Mini App.
- O usuario pedir painel administrativo.
- O usuario pedir SaaS.
- O usuario pedir demo comercial.
- O usuario pedir qualquer interface visual para cliente.

## Referencias possiveis

Procurar e considerar, quando disponivel:

- Site do cliente.
- Instagram.
- Logo.
- Paleta de cores.
- Fotos do estabelecimento.
- Prints do WhatsApp ou atendimento.
- Materiais graficos.
- Referencias enviadas pelo cliente.
- Concorrentes.
- Nicho do negocio.
- Publico-alvo.

## Processo obrigatorio

1. Identificar o tipo de negocio.
2. Analisar as referencias visuais disponiveis.
3. Extrair cores principais e secundarias.
4. Identificar o estilo visual.
5. Definir a direcao visual antes de codar.
6. Aplicar essa direcao na interface.

## Estilos visuais

Classificar a direcao visual com uma ou mais categorias:

- premium
- popular
- tecnico
- moderno
- minimalista
- agressivo
- institucional
- jovem
- automotivo
- luxo
- local/comercial

## Regras de design

- Nao usar paleta generica sem motivo.
- Nao repetir sempre azul, roxo ou verde.
- Nao criar interface com cara de template padrao de IA.
- Nao usar gradiente excessivo se a marca nao pedir.
- Nao usar cards genericos demais.
- Usar espacamento consistente.
- Criar hierarquia visual clara.
- Adaptar botoes, badges, cards, tabelas e sidebar ao estilo do cliente.
- Manter responsividade.
- Manter visual profissional e vendavel.
- Nao mudar a logica do sistema sem necessidade.

## Quando faltar referencia visual

Se nao houver referencia visual suficiente:

- Criar uma direcao visual provisoria baseada no nicho.
- Explicar que a identidade pode melhorar se o cliente enviar logo, Instagram, site ou fotos.
- Evitar escolhas muito marcantes sem base.
- Preferir visual profissional, claro e ajustavel.

## Saida esperada antes de codar

Antes de implementar ou alterar a interface, registrar:

- Tipo de negocio
- Referencias analisadas
- Cores principais
- Cores secundarias
- Estilo visual escolhido
- Direcao visual
- Elementos de UI que serao adaptados
- O que sera evitado para nao parecer template

## Checklist antes de finalizar

- A interface parece pertencer ao cliente?
- As cores fazem sentido com a marca?
- O estilo combina com o nicho?
- Parece produto real, nao template?
- Da vontade de mandar print para vender?
- A hierarquia esta clara?
- O mobile funciona bem?
- O design esta diferente de outros projetos anteriores?

## Integracao com outras skills da Capta
