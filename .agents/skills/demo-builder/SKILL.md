# Demo Builder

## Objetivo

Criar demos comerciais rapidas e vendaveis para qualquer nicho.

## Use quando

- O usuario quiser impressionar um lead.
- O usuario pedir um prototipo.
- O usuario precisar mandar print ou video para vender.
- O usuario quiser validar uma ideia antes de construir o sistema real.

## Regras

- A demo deve parecer profissional em ate 30 segundos.
- Nao criar backend real sem necessidade.
- Usar dados fake realistas.
- Fazer interface bonita, responsiva e facil de apresentar.
- Preparar para Vercel quando for web.
- Criar mensagem curta para enviar ao cliente junto com a demo.
- Deixar claro no README que e demo visual, quando for o caso.
- Separar comportamento simulado de funcionalidade real.
- Nunca usar tokens, senhas, credenciais reais ou dados sensiveis.

## Processo

1. Definir a decisao comercial que a demo precisa gerar.
2. Identificar o nicho, a dor principal e a historia que a demo vai contar.
3. Criar o menor fluxo visual que impressiona e comunica valor.
4. Usar dados fake com nomes, status, valores, datas e mensagens plausiveis.
5. Implementar interface responsiva, com estados claros e visual profissional.
6. Evitar backend real, banco ou autenticacao se mock/local storage resolver.
7. Adicionar README com objetivo, como rodar, como testar e limites da demo.
8. Incluir uma mensagem curta para envio ao lead.

## Saida esperada

Ao criar uma demo, entregar:

- Projeto funcionando
- README
- Dados fake realistas
- Instrucoes de teste
- Mensagem de envio para o lead
- Checklist de apresentacao
- Proximos passos para transformar em sistema real

## Padroes de demo

- Login pode ser simulado se nao for parte central da venda.
- Pagamento deve ser mockado, a menos que o usuario peca integracao real.
- WhatsApp, Telegram, email e webhooks podem ser representados por telas, logs ou estados visuais.
- Dashboards podem usar metricas fake, desde que o README explique.
- A primeira tela deve mostrar valor, nao uma pagina de marketing vazia.
